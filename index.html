An Introduction to WSO2 Configurator

Product configurations can be automated with orchestration management systems such as Puppet, Chef, Salt, Ansible, etc. Most of these orchestration systems provide their own templating engine for templating configuration files. The problem with this model is that for each orchestration system a separate set of templates need to be created for the same product. Switching between different orchestration systems & maintaining different sets of templates files are very costly. WSO2 Configurator was introduced to solve this problem by implementing a generic templating solution which can work with any orchestration system: WSO2 Configurator is a python module written using Jinja2 template engine which can configure a product using a set of key/value pairs. »
